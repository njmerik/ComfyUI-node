# Модернизация Wan2Video workflow

Я взяла стандартный шаблон генерации видео из ComfyUI и значительно его улучшила.  
Этот воркфлоу показывает мои навыки работы с **ComfyUI, кастомными нодами, GGUF-моделями и пост-обработкой**.

## 🚀 Что было сделано

### 1. Переход на GGUF-модели
- Заменила обычные `.safetensors` на **GGUF‑квантованные модели** (`Artius‑Wan22‑14b‑I2V‑high/low‑Q4_K_M‑v2.gguf`)
- Это снижает потребление VRAM и ускоряет загрузку без потери качества

### 2. Добавлен апскейлинг 
- Интегрировала **Ultimate SD Upscale** — интеллектуальное увеличение изображения
- Первичный апскейл в **1.7x** через `ImageScaleBy`
- Финальный проход с `denoise 0.3` для сохранения деталей

### 3. Профессиональный видеовыход
- Заменила стандартный `SaveVideo` на **VHS_VideoCombine**
- Настройка `crf: 16` — оптимальное качество/размер
- Сохранение в MP4 с метаданными

### 4. Оптимизация пайплайна семплинга
- Два прохода KSampler с **разделением шума**
- Тонкая настройка шагов (4 → 2 → до конца)
- `CFG = 1` для более аккуратной генерации

### 5. Улучшение структуры
- Вынесла **seed** в отдельную ноду PrimitiveInt
- Чёткое разделение позитивного и негативного промптов
- Логичное именование нод

### 6. Референсная генерация
- Использован стартовый кадр (`cat_ref.png`)
- Генерация видео с опорой на изображение

## 🛠 Использованные инструменты
- ComfyUI + кастомные ноды:
  - `WanFirstLastFrameToVideo`
  - `UltimateSDUpscaleNoUpscale`
  - `VHS_VideoCombine`
- GGUF-модели (квантованные)
- Управление через примитивы
